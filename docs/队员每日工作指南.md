## 二、每天早上（2 分钟）

### 第 1 步：用 IDEA 打开项目

```
File → Open → 选 ~/Desktop/zwpt001 目录 → OK
等 IDEA 右下角的进度条跑完
```

### 第 2 步：拉取最新代码

在 IDEA 底部工具栏点 **Terminal**，输入：

```
git pull origin main
```

> 或者用 IDEA 顶部菜单：Git → Pull → 选 origin/main → Pull

### 第 3 步：切到自己的分支

```
git checkout feature/member-x
```

> 成员A = `feature/member-a`，成员B = `feature/member-b`，以此类推

---

## 三、启动服务（IDEA 一键启动）

### 方法：找到 Application 类 → 点绿色三角形

```
1. 在 IDEA 左侧项目树里展开你的服务目录
   例如：gov-user-service → src → main → java → com.gov.user

2. 找到 XXXApplication.java（带 main 方法的类）
   例如：UserApplication.java

3. 点行号左边的 ▶ 绿色三角形 → Run 'UserApplication'

4. 等 IDEA 底部 Run 窗口出现：
   Started UserApplication in X seconds   ← 启动成功！
```

### 你有两个服务？再开一个 Run

```
第一个服务启动后 → 在另一个 Application 类上右键 → Run
IDEA 会新开一个 Run 标签，两个服务同时运行。
```

### 成员B 特别注意

你需要同时启动 reception-service 和 open-service，两个都要跑。

## 四、对 AI 说需求（核心步骤）

### 4.1 在 IDEA 的 AI 对话框里输入（每次都复制这段）

```
我正在开发 gov-xxx-service，端口 8xxx。
项目规则文件在 .trae/rules/project_rules.md，请先读取。

【我的需求】
（这里写你要做什么功能）

参考已有代码：
- Controller 模板：gov-user-service 的 UserController
- Service 模板：gov-user-service 的 UserServiceImpl
- Entity 模板：gov-user-service 的 UserEntity
- YAML 模板：gov-user-service/src/main/resources/application-dev.yml

生成代码后：
1. 告诉我在 IDEA 里怎么验证编译通过
2. 告诉我如何在 doc.html 上测试
```

### 4.2 常见需求直接替换中间那段

| 需求 | 对 AI 说的内容 |
|------|---------------|
| 新增接口 | "帮我新增一个 xxx 接口，入参是 a/b/c，返回 d/e/f" |
| 修改接口 | "帮我把 xxx 接口的返回值加一个字段 yyy" |
| 加数据库表 | "帮我新建一张 t_xxx 表，字段有 a、b、c，同时更新 Entity 和 SQL 文件" |
| 加查询条件 | "帮我在 xxx 分页查询里加一个根据 yyy 筛选的条件" |
| 调用其他服务 | "我需要调用 gov-user-service 的 getById 接口，帮我在 Service 里写调用代码" |

### 4.3 写 Feign 调用其他服务（成员B、成员C 看）

你的项目里已经有写好的 Feign 接口，AI 会自动注入，你什么都不用管。

## 六、编译验证

AI 生成代码后，在 IDEA 里验证：

**方法 1（推荐）**：顶部菜单 **Build → Rebuild Project**

**方法 2（如果只想编译自己的模块）**：右侧边栏点 **Maven** → 展开你的模块 → Lifecycle → 双击 **compile**

- Build Successful → 通过
- Build Failed → 截图底部 Build 窗口的错误信息 → 粘贴给 AI → "编译报错了，帮我修"

## 七、每天下班前（提交代码）

### 用 IDEA 提交（最简单）

```
1. 顶部菜单 Git → Commit（或按 ⌘K）
2. 确认改动的文件都是你自己的模块（没有改别人的）
3. Commit Message 写：feat(模块): 做了什么功能
4. 点 Commit and Push
```

Commit 消息格式：
```
feat(complaint): 新增投诉工单查询接口
fix(item): 修复事项分页查询缺少删除过滤
```

### 去 GitHub 创建 PR

1. 浏览器打开 `https://github.com/Liu3389/gov-platform`
2. 点 **Pull requests** → **New pull request**
3. base 选 `develop`，compare 选你的分支 `feature/member-x`
4. 点 **Create pull request**
5. 标题写清楚做了什么 → 点 **Create**

然后等组长审批合并。

## 八、遇到问题怎么办

| 问题 | 怎么做 |
|------|--------|
| IDEA 红色波浪线 | 右键 pom.xml → Maven → Reload Project |
| 启动报错 | 复制 Run 窗口的红色报错 → 粘贴给 AI → "帮我分析启动报错" |
| 接口返回 500 | 把浏览器上的返回结果复制给 AI → "这个接口报错了" |
| 端口被占用 | 去 Run 窗口点红色方块停止之前的服务，再重新 Run |
| 不知道端口号 | 看本文档第一节 |
| AI 说要改 gov-common | ❌ 停下来，告诉组长，组长来改 |
| AI 说要改别人的模块 | ❌ 停下来，告诉组长 |
| AI 说要加新表 | ✅ 可以，但 PR 时提醒组长审核 |
| 不确定能不能做 | 问组长 |

## 九、规则铁律（AI 和人都必须遵守）

1. ❌ **绝对不能改别人的模块文件**（gov-common 只有组长能改）
2. ❌ **绝对不能直接 push 到 main 或 develop**
3. ❌ **绝对不能用 `@Autowired`**（用 `@RequiredArgsConstructor`）
4. ❌ **绝对不能用 `java.util.Date`**（用 `LocalDateTime`）
5. ✅ 写操作必须加 `@Log` 注解
6. ✅ 查询必须加 `.eq(Deleted, 0)` 过滤已删除数据
7. ✅ Controller 返回值必须用 `Result<T>` 包装
8. ✅ 新增表时必须同步提交 SQL 文件到 `docker/mysql/init/`

## 十、AI 专用指令（你的 AI 每次都要读）

```
【AI 指令 — 每条规则必须遵守】

1. 项目主规则：.trae/rules/project_rules.md
   包含：技术栈版本、包结构、禁止清单、端口分配

2. 代码任务规则（按需读取）：
   - 写 Controller：.trae/rules/tasks/controller.md
   - 写 Service/Mapper：.trae/rules/tasks/service.md
   - 写 Entity/建表：.trae/rules/tasks/sql.md
   - 写 Feign 调用：.trae/rules/tasks/feign.md
   - 写 YAML 配置：.trae/rules/tasks/yaml.md

3. 生成代码后必须做的事：
   - 检查是否使用了 @Autowired → 必须用 @RequiredArgsConstructor + final
   - 检查是否使用了 java.util.Date → 必须用 LocalDateTime
   - 检查 Entity 是否继承 BaseEntity
   - 检查 Controller 是否有 @Tag @Operation
   - 检查写操作是否有 @Log
   - 检查查询是否包含 .eq(Deleted, 0)

4. 如果你的代码需要修改 gov-common：
   → 不要改！告诉用户"联系组长，只有组长能改 gov-common"

5. 如果你需要修改其他服务的代码：
   → 不要改！告诉用户"这是 XX 成员的模块，请通过组长协调"
```
