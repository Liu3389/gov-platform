# 环境搭建指南 —— 从零到启动

> 适用操作系统：**Windows 10/11** 和 **macOS**（本文档覆盖两套系统）。
> 预计耗时：首次 30-60 分钟（主要花在下载 Docker 镜像上）。

---

## 一、安装必备软件

### 1.1 JDK 17

| 系统 | 操作 |
|------|------|
| **Windows** | 下载 [JDK 17 MSI 安装包](https://download.oracle.com/java/17/archive/jdk-17.0.10_windows-x64_bin.msi)，双击安装，一路下一步。 |
| **macOS** | 下载 [JDK 17 DMG 安装包](https://download.oracle.com/java/17/archive/jdk-17.0.10_macos-aarch64_bin.dmg) (Apple Silicon) 或 [x64 版本](https://download.oracle.com/java/17/archive/jdk-17.0.10_macos-x64_bin.dmg) (Intel Mac)，双击安装。 |

**验证安装**（终端/CMD 里执行）：

```
java -version
```
看到 `17.0.x` 即成功。

---

### 1.2 Maven 3.8+

| 系统 | 操作 |
|------|------|
| **Windows** | 下载 [Maven zip](https://dlcdn.apache.org/maven/maven-3/3.9.6/binaries/apache-maven-3.9.6-bin.zip)，解压到 `C:\tools\apache-maven`。然后把 `C:\tools\apache-maven\bin` 加入系统环境变量 PATH。 |
| **macOS** | 终端执行 `brew install maven`。如果没装 Homebrew，先去 [brew.sh](https://brew.sh) 安装。 |

**验证：**

```
mvn -version
```
看到 `3.9.x` + `Java 17` 即成功。

### 1.3 Docker Desktop

> Docker 是必须要的，项目的 MySQL、Redis、Nacos、RabbitMQ、Nginx 全部用 Docker 运行。

| 系统 | 操作 |
|------|------|
| **Windows** | 下载 [Docker Desktop for Windows](https://www.docker.com/products/docker-desktop/)，安装后重启电脑。**注意：Windows 必须开启 WSL2**（安装程序会引导你）。安装后在托盘里看到 Docker 鲸鱼图标变绿才算启动成功。 |
| **macOS** | 下载 [Docker Desktop for Mac](https://www.docker.com/products/docker-desktop/)，双击安装，启动后在菜单栏看到鲸鱼图标变绿即成功。 |

**配置镜像加速（国内必做）**：

打开 Docker Desktop → 设置齿轮 ⚙ → `Docker Engine` → 在 JSON 中添加：

```json
"registry-mirrors": [
  "https://docker.1ms.run",
  "https://docker.xuanyuan.me"
]
```

点 `Apply & Restart`。

## 三、启动基础环境（Docker 一键启动）

打开终端（Windows 用 CMD 或 Git Bash、macOS 用终端），依次执行：

```bash
# 进入 Docker 目录
cd docker

# 一键启动所有中间件（MySQL、Redis、Nacos、RabbitMQ、Nginx）
docker-compose up -d

# 查看是否全部启动成功
docker-compose ps
```

看到 5 个服务都是 `Up` 状态就成功了。

| 服务 | 端口 | 说明 |
|------|:---:|------|
| MySQL | 3307 | 已自动建好所有数据库和表，用户名/密码: `root/root123456` |
| Redis | 6379 | 缓存 |
| Nacos | 8848 | 注册配置中心，控制台 http://localhost:8848/nacos 用户名/密码: `nacos/nacos` |
| RabbitMQ | 5672 | 消息队列，管理界面 http://localhost:15672 用户名/密码: `guest/guest` |
| Nginx | 80 | 前端 + API 代理 |

> **首次启动要下载镜像（3-5分钟），之后启动只需几秒。**

## 四、导入 IDEA 并启动服务

### 4.1 导入项目

1. 打开 IDEA → `File → Open`
2. 选择项目目录下的 **`gov-platform-parent`** 文件夹（注意不是外层 `zwpt001`）
3. IDEA 会自动下载 Maven 依赖（右下角有进度条，等它跑完）
4. 打开右侧 **Maven 面板**，应该能看到 13 个模块

### 4.2 启动顺序

**按顺序启动，前一个启动成功后再启动下一个：**

| 顺序 | 服务 | 端口 | 启动类 |
|:---:|------|:---:|------|
| 1 | gov-gateway | 8091 | `GatewayApplication` |
| 2 | gov-user-service | 8081 | `UserApplication` |
| 3 | gov-item-service | 8092 | `ItemApplication` |
| 4 | gov-reception-service | 8083 | `ReceptionApplication` |
| 5 | gov-activiti-service | 8084 | `ActivitiApplication` |
| 6 | gov-license-service | 8085 | `LicenseApplication` |
| 7 | gov-complaint-service | 8086 | `ComplaintApplication` |
| 8 | gov-open-service | 8087 | `OpenApplication` |
| 9 | gov-datashare-service | 8088 | `DatashareApplication` |
| 10 | gov-message-service | 8089 | `MessageApplication` |
| 11 | gov-monitor-service | 8090 | `MonitorApplication` |

**启动方法**：找到启动类（如 `com.gov.gateway.GatewayApplication`），右键 → `Run 'GatewayApplication'`。

> 日常开发只需要启动 **网关 + 自己负责的服务 + 依赖的服务** 即可，不用全部启动。

### 4.3 验证启动成功

- 浏览器打开 Nacos：`http://localhost:8848/nacos` → 服务管理 → 服务列表，能看到已启动的服务名
- 浏览器打开 Knife4j 网关文档：`http://localhost:8091/doc.html`，能看到所有已启动服务的接口

## 五、分工与开发

### 5.1 四人分工表

| 角色 | 负责服务 | 端口 | 需要启动的依赖 |
|:---:|------|:---:|------|
| **A** | gov-user-service、gov-item-service | 8081/8092 | MySQL、Redis、Nacos |
| **B** | gov-reception-service、gov-open-service | 8083/8087 | MySQL、Redis、Nacos、gov-user-service |
| **C** | gov-activiti-service、gov-license-service | 8084/8085 | MySQL、Redis、Nacos、gov-user-service |
| **D** | gov-complaint-service、gov-datashare-service、gov-message-service | 8086/8088/8089 | MySQL、Redis、Nacos、RabbitMQ |

### 5.2 日常开发流程

```
1. docker-compose up -d                          ← 启动中间件（只需一次）
2. 启动自己负责的服务（在 IDEA 里 Run）            ← 启动自己的服务
3. 打开 http://localhost:8091/doc.html             ← 查看接口契约
4. 写代码
5. 在 Knife4j 中测试自己的接口
6. mvn compile（或 IDEA Build）验证通过
7. git commit -m "feat(xxx): xxx"                 ← 提交代码
```

### 5.3 关闭环境

```bash
cd docker
docker-compose down      # 停止容器（数据保留）
docker-compose down -v   # 停止容器并删除数据（重来用）
```

## 六、常见问题

### Q1：IDEA 启动报 `com.mysql.cj.jdbc.exceptions.CommunicationsException`

**原因**：MySQL 容器没启动或端口被占用。
**解决**：`docker-compose ps` 确认 gov-mysql 是 `Up` 状态。

### Q2：启动报 `Connection refused` 连接 Nacos

**原因**：Nacos 容器还没完全启动。
**解决**：Nacos 启动较慢（约30秒），等 `docker-compose logs -f nacos` 看到 `Nacos started successfully` 后再启动微服务。

### Q3：IDEA 编译报红但 mvn compile 正常

**解决**：`File → Invalidate Caches → Invalidate and Restart`。

### Q4：端口被占用

**解决**：在 `application-dev.yml` 里修改 `server.port` 为其他端口。

### Q5：前端 `gov-web` 怎么启动

```bash
cd gov-web
npm install          # 首次需要安装依赖
npm run serve        # 开发模式启动，默认端口 8088
```

### Q6：Windows 和 Mac 的配置文件有差异吗

**没有差异。** 所有 `application-dev.yml` 里中间件地址都是 `localhost`，两个系统通用。Docker 也提供了统一的 docker-compose.yml。
