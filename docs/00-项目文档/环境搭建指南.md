# 环境搭建指南

> 本团队无公共服务器，每位成员（Mac/Windows）的电脑即是一套完整、隔离的微服务开发环境。本指南说明如何搭建本地环境、启动中间件、调试服务。

***

## 一、环境准备清单

### 1.1 统一换行符

```bash
# 所有成员执行
git config --global core.autocrlf false
git config --global core.eol lf
```

### 1.2 内存建议

- 电脑内存：16G 起步。
- Docker Desktop：至少 4G 内存、2 核。
- 单个 JVM 服务：`-Xms256m -Xmx512m`。
- 同时启动服务数：按需，一般 2–4 个。

## 二、安装 Docker Desktop

### 2.1 Mac

1. 下载 Apple Silicon 或 Intel 版本。
2. 安装并启动，Settings → Resources，建议 CPU 4 核、Memory 8G。

### 2.2 Windows

1. 下载 Windows 版 Docker Desktop。
2. 选择 **Use WSL 2 instead of Hyper-V**。
3. 确保 WSL2 启用：
   ```powershell
   wsl --install
   wsl --set-default-version 2
   ```
4. Docker Desktop → Settings → Resources → WSL integration，勾选你的 WSL 发行版。

### 2.3 验证

```bash
docker --version
docker-compose --version
```

## 三、Docker 中间件一键启动

### 3.1 目录结构

```text
docker/
├── docker-compose.yml
├── mysql/
│   └── init/
│       ├── 01-init-db.sql
│       └── 02-init-data.sql
├── redis/
│   └── redis.conf
├── nacos/
│   └── nacos_config_export.zip
└── nginx/
    └── nginx.conf
```

### 3.2 docker-compose.yml

```yaml
version: '3.8'

services:
  mysql:
    image: mysql:8.0.33
    container_name: zwpt-mysql
    environment:
      MYSQL_ROOT_PASSWORD: 123456
      MYSQL_DATABASE: gov_user
    ports:
      - "3306:3306"
    volumes:
      - ./mysql/data:/var/lib/mysql
      - ./mysql/init:/docker-entrypoint-initdb.d
      - ./mysql/conf:/etc/mysql/conf.d
    command: --default-authentication-plugin=mysql_native_password

  redis:
    image: redis:7-alpine
    container_name: zwpt-redis
    ports:
      - "6379:6379"
    volumes:
      - ./redis/data:/data
      - ./redis/redis.conf:/usr/local/etc/redis/redis.conf

  nacos:
    image: nacos/nacos-server:v2.2.3
    container_name: zwpt-nacos
    environment:
      MODE: standalone
      SPRING_DATASOURCE_PLATFORM: mysql
      MYSQL_SERVICE_HOST: mysql
      MYSQL_SERVICE_DB_NAME: nacos
      MYSQL_SERVICE_PORT: 3306
      MYSQL_SERVICE_USER: root
      MYSQL_SERVICE_PASSWORD: 123456
    ports:
      - "8848:8848"
      - "9848:9848"
    depends_on:
      - mysql

  nginx:
    image: nginx:1.25-alpine
    container_name: zwpt-nginx
    ports:
      - "80:80"
    volumes:
      - ./nginx/nginx.conf:/etc/nginx/nginx.conf
      - ./nginx/html:/usr/share/nginx/html
```

### 3.3 MySQL 初始化脚本

`docker/mysql/init/01-init-db.sql`：

```sql
CREATE DATABASE IF NOT EXISTS gov_user DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE DATABASE IF NOT EXISTS gov_item DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE DATABASE IF NOT EXISTS gov_reception DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE DATABASE IF NOT EXISTS gov_activiti DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE DATABASE IF NOT EXISTS gov_license DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE DATABASE IF NOT EXISTS gov_complaint DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE DATABASE IF NOT EXISTS gov_open DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE DATABASE IF NOT EXISTS gov_datashare DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE DATABASE IF NOT EXISTS gov_message DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE DATABASE IF NOT EXISTS gov_monitor DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE DATABASE IF NOT EXISTS nacos DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

`docker/mysql/init/02-init-data.sql`：写入字典、初始管理员等数据。

### 3.4 redis.conf

```conf
requirepass 123456
appendonly yes
```

### 3.5 nginx.conf

```nginx
server {
    listen 80;
    server_name localhost;

    location / {
        root /usr/share/nginx/html;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://host.docker.internal:8080/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

> Windows/Mac 均使用 `host.docker.internal` 访问宿主机 Gateway。

## 五、后端本地调试（按需启动）

### 5.1 启动顺序

1. Docker 中间件（Nacos、MySQL、Redis、Nginx）。
2. `gov-gateway` (8080)。
3. `gov-user-service` (8081)（如果网关需要鉴权）。
4. 你负责的业务服务。

### 5.2 IDEA 启动配置

| 配置项             | 值                                         |
| --------------- | ----------------------------------------- |
| Active profiles | `dev`                                     |
| VM options      | `-Dfile.encoding=UTF-8 -Xms256m -Xmx512m` |

### 5.3 验证 Knife4j

- 网关聚合文档：`http://localhost:8080/doc.html`
- 单服务文档：`http://localhost:{服务端口}/doc.html`

### 5.4 Feign 本地调用

1. 确认被调用服务已注册到 Nacos（`http://localhost:8848/nacos`）。
2. 检查服务名是否一致（`spring.application.name`）。
3. 检查 Feign 接口路径是否正确。

## 七、常见问题速查

| 问题              | 可能原因                    | 解决                                                           |
| --------------- | ----------------------- | ------------------------------------------------------------ |
| 服务启动后 Nacos 看不到 | 注册中心地址错误                | 检查 `spring.cloud.nacos.discovery.server-addr=127.0.0.1:8848` |
| Feign 调用 404    | 服务未启动 / 路径写错            | 检查 Nacos 服务列表和 `@FeignClient` name                           |
| 前端请求跨域          | 网关未配置 CORS              | 在 Gateway 添加 CorsWebFilter                                   |
| MySQL 连接超时      | Docker MySQL 没启动 / 库未创建 | `docker ps` + `show databases;`                              |
| Redis 连不上       | 密码错误 / db 号冲突           | 检查 `spring.redis.password` 和 `database`                      |
| Knife4j 打不开     | 服务没启动 / 网关路由未配          | 检查网关路由和服务注册                                                  |
| Mac 端口被占用       | 系统服务占用                  | `lsof -i :3306` 后 `kill -9 PID`                              |
| Windows 端口被占用   | 本机已装 MySQL              | 停止服务或修改 docker-compose 端口映射                                  |

***

## 八、本地联调流程

1. 后端启动自己负责的服务 + gateway + user-service。
2. 访问 `http://localhost:8080/doc.html` 做 Knife4j 自测。
3. 群里发送：已启动服务清单 + 可测接口 + 截图。
4. 前端通过 `http://localhost` 或 `npm run dev` 调用真实接口。
5. 发现问题后，双方同时用 AI 分析 Knife4j 请求/响应/日志。
