### 2.1 新成员必读


## 四、AI 开发使用指南

### 4.1 AI Prompt 模板使用流程

```
1. 打开 AI编码Prompt模板.md
2. 找到对应的任务类型（如：生成 Entity、生成 Controller、生成 BPMN）
3. 复制 Prompt 模板
4. 填入具体参数（如：表名、字段、服务名）
5. 发给 AI，AI 按约束生成代码
6. 人工 Review 后合并
```

### 4.2 各文档的 AI Prompt 模板位置

| 文档              | Prompt 模板位置              |
| --------------- | ------------------------ |
| 数据库设计文档.md      | 第四章：AI 生成数据库代码 Prompt 模板 |
| 工作流设计文档.md      | 第六章：AI 生成工作流代码 Prompt 模板 |
| 业务流程说明.md       | 各场景详解中的 AI Prompt 模板     |
| AI编码Prompt模板.md | 全文（核心入口）                 |

### 4.3 AI 必须遵守的约束

| 约束类型                 | 来源文档       |
| -------------------- | ---------- |
| 命名规范（表名、类名、方法名）      | 开发规范.md    |
| 建表规范（主键、必备字段、索引）     | 数据库设计文档.md |
| 接口规范（路径、返回体、Knife4j） | 接口规范.md    |
| 流程规范（流程Key、变量名、监听器）  | 工作流设计文档.md |
| 业务约束（状态枚举、流程变量）      | 业务流程说明.md  |
