# 数据库开发规范

> 本文档是 AI 和组员编写数据库代码的**铁律规范**，必须严格遵守，避免数据库开发混乱。
> 
> **AI 使用时机**：当需要新建表、修改表结构、编写 Entity 类时，必须同时读取：
> - `.trae/rules/project_rules.md`（主规则）
> - `.trae/rules/tasks/sql.md`（SQL 文件管理流程）
> - 本文档（表结构设计规范）

---

## 一、核心原则（铁律）

| 铁律 | 说明 | 违反后果 |
|------|------|----------|
| **禁止手动改表** | 所有表结构变更必须通过 SQL 脚本，禁止用 Navicat 等工具手动改表不提交 SQL | 代码审查驳回 |
| **核心表变更审批** | 核心主表（用户、办件、流程、证照）的删字段、改类型、改关联必须组长审批 | 强制回滚 |
| **统一继承 BaseEntity** | 所有业务表必须继承 BaseEntity（id、createTime、updateTime、createBy、updateBy、deleted） | 编译报错 |
| **敏感字段标注脱敏** | 身份证、手机号等敏感字段必须在 SQL 注释里标注 `[脱敏]` | 安全审查驳回 |
| **脚本化提交** | 所有 SQL 变更必须和代码一起提交，SQL 文件放在 `docker/mysql/init/` | 不予合并 |
| **禁止修改已有 SQL** | 已有 SQL 文件不可修改（因为只在 MySQL 容器首次创建时执行一次），改表用 ALTER 文件 | 强制回滚 |

## 二、表结构规范

### 2.1 必备字段（所有表必须包含）

```sql
`id` BIGINT NOT NULL COMMENT '主键（雪花ID）',
`create_time` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
`update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
`create_by` BIGINT COMMENT '创建人ID',
`update_by` BIGINT COMMENT '更新人ID',
`deleted` TINYINT DEFAULT 0 COMMENT '逻辑删除：0正常 1删除',
```

**说明**：
- `id`：使用 MyBatis-Plus `IdType.ASSIGN_ID`（雪花算法）
- `deleted`：逻辑删除字段，MyBatis-Plus 配置 `@TableLogic`
- 时间字段：使用 `DATETIME`，默认值 `CURRENT_TIMESTAMP`

### 2.2 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 表名 | `t_` + 业务名（下划线） | `t_user_info`、`t_item_base`、`t_reception_record` |
| 字段名 | 小写字母 + 下划线 | `user_name`、`apply_no`、`create_time` |
| 主键 | `id` | `id BIGINT NOT NULL` |
| 索引 | `idx_` + 字段名 或 `idx_` + 字段1_字段2 | `idx_user_id`、`idx_apply_no`、`idx_dept_status` |
| 唯一索引 | `uk_` + 字段名 | `uk_username`、`uk_item_code`、`uk_apply_no` |

**注意**：表名不需要服务前缀（如 `t_user_info` 而非 `t_user_user_info`），因为每个服务独立数据库。

### 2.3 字段类型规范

| 数据类型 | MySQL 类型 | 说明 | 示例 |
|----------|------------|------|------|
| 主键 | `BIGINT` | 雪花 ID（MyBatis-Plus `IdType.ASSIGN_ID`） | `id BIGINT NOT NULL` |
| 状态 | `TINYINT` | 0/1/2/3，配合枚举类，必须在注释说明含义 | `status TINYINT DEFAULT 0 COMMENT '状态：0待审 1通过 2驳回'` |
| 名称 | `VARCHAR(50~100)` | 用户名、事项名、部门名 | `user_name VARCHAR(50)` |
| 编码 | `VARCHAR(20~50)` | 唯一编码，加唯一索引 | `item_code VARCHAR(50)` |
| 内容 | `TEXT` | 长文本、JSON 数据 | `content TEXT` |
| 时间 | `DATETIME` | 默认值 `CURRENT_TIMESTAMP` | `create_time DATETIME DEFAULT CURRENT_TIMESTAMP` |
| 金额 | `DECIMAL(10,2)` | 精确计算，不用 FLOAT | `amount DECIMAL(10,2)` |
| 手机号 | `VARCHAR(20)` | 支持国际号码 | `phone VARCHAR(20)` |
| 身份证 | `VARCHAR(100)` | 加密存储，注释标注 `[脱敏][SM4加密]` | `id_card VARCHAR(100)` |
| 布尔值 | `TINYINT(1)` | 0/1，注释说明含义 | `is_enabled TINYINT(1) DEFAULT 1 COMMENT '是否启用：0否 1是'` |

### 2.4 敏感字段标注规范

```sql
-- 正确示例
`id_card` VARCHAR(100) COMMENT '身份证号 [脱敏][SM4加密存储]',
`phone` VARCHAR(20) COMMENT '手机号 [脱敏][展示时掩码138****8000]',
`password` VARCHAR(100) COMMENT '密码 [脱敏][BCrypt哈希]',

-- 错误示例（缺少标注）
`id_card` VARCHAR(100) COMMENT '身份证号',  -- ❌ 缺少 [脱敏] 标注
```

**脱敏规则**：
- 身份证号：展示时掩码 `320***********1234`
- 手机号：展示时掩码 `138****8000`
- 密码：BCrypt 哈希，永不展示
- CA 证书内容：SM4 加密，永不展示

## 三、建表 SQL 模板

### 3.1 标准建表模板

```sql
CREATE TABLE IF NOT EXISTS `t_xxx` (
  -- ==================== 必备字段 ====================
  `id` BIGINT NOT NULL COMMENT '主键（雪花ID）',
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `create_by` BIGINT COMMENT '创建人ID',
  `update_by` BIGINT COMMENT '更新人ID',
  `deleted` TINYINT DEFAULT 0 COMMENT '逻辑删除：0正常 1删除',
  
  -- ==================== 业务字段 ====================
  `xxx_code` VARCHAR(50) NOT NULL COMMENT '编码（唯一）',
  `xxx_name` VARCHAR(100) NOT NULL COMMENT '名称',
  `status` TINYINT DEFAULT 0 COMMENT '状态：0待审 1通过 2驳回',
  `remark` VARCHAR(200) COMMENT '备注',
  
  -- ==================== 主键和索引 ====================
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_xxx_code` (`xxx_code`),
  KEY `idx_status` (`status`),
  KEY `idx_create_time` (`create_time`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='表说明';
```

### 3.2 关联表模板

```sql
CREATE TABLE IF NOT EXISTS `t_xxx_relation` (
  `id` BIGINT NOT NULL COMMENT '主键',
  `main_id` BIGINT NOT NULL COMMENT '主表ID',
  `sub_id` BIGINT NOT NULL COMMENT '子表ID',
  `relation_type` TINYINT DEFAULT 1 COMMENT '关联类型：1主办 2协办',
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
  `deleted` TINYINT DEFAULT 0,
  PRIMARY KEY (`id`),
  KEY `idx_main_id` (`main_id`),
  KEY `idx_sub_id` (`sub_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='关联表';
```

## 四、索引规范

### 4.1 必须加索引的场景

| 场景 | 索引类型 | 示例 |
|------|----------|------|
| 唯一编码 | UNIQUE | `uk_item_code`、`uk_apply_no` |
| 外键关联 | INDEX | `idx_user_id`、`idx_dept_id`、`idx_item_id` |
| 状态查询 | INDEX | `idx_status` |
| 时间范围查询 | INDEX | `idx_create_time`、`idx_update_time` |
| 高频组合查询 | INDEX（联合） | `idx_dept_status`、`idx_user_create_time` |

### 4.2 索引命名规范

```sql
-- 单字段索引
KEY `idx_user_id` (`user_id`)

-- 联合索引（字段顺序按查询频率排列）
KEY `idx_dept_status` (`dept_id`, `status`)

-- 唯一索引
UNIQUE KEY `uk_username` (`username`)
```

### 4.3 索引设计原则

1. **最左前缀原则**：联合索引 `(a, b, c)` 可以支持 `a`、`a+b`、`a+b+c` 的查询
2. **避免过多索引**：单表索引不超过 5 个，否则影响写入性能
3. **区分度高的字段优先**：联合索引中，区分度高的字段放在前面

## 五、开发流程规范

### 5.1 新增表流程

```
1. 组员开发新功能，需要新建表
2. 参考《数据库设计文档.md》中的表结构
3. 使用本文档的建表模板编写 SQL
4. 本地执行 SQL 验证结构正确
5. 将 SQL 文件放到 docker/mysql/init/ 目录（使用未占用的序号 12-98）
6. 代码和 SQL 一起提交到 Git
7. 组长审核 SQL 规范（命名、索引、注释、脱敏）
8. 合并入主分支
9. 全队 docker-compose down && docker-compose up -d mysql 重建
```

### 5.2 修改表流程

```
1. 组员发现需要加字段/改字段
2. 编写 ALTER TABLE SQL（不是重建表）
3. 本地执行验证
4. 将 SQL 放到 docker/mysql/init/XX-alter.sql（例如 13-alter-user-add-nickname.sql）
5. 代码和 SQL 一起提交
6. 核心主表变更需组长审批
7. 合并入主分支
8. 全队 docker-compose down && docker-compose up -d mysql 重建
```

**注意**：永远不修改已有 SQL 文件（01-11、99），因为它们只在 MySQL 容器首次创建时执行一次。

### 5.3 ALTER TABLE 示例

```sql
-- 新增字段
ALTER TABLE `t_user_info` 
ADD COLUMN `nickname` VARCHAR(50) COMMENT '昵称' AFTER `real_name`;

-- 新增索引
ALTER TABLE `t_user_info` 
ADD INDEX `idx_nickname` (`nickname`);

-- 修改字段（谨慎）
ALTER TABLE `t_user_info` 
MODIFY COLUMN `phone` VARCHAR(30) COMMENT '手机号 [脱敏][展示时掩码]';

-- 删除字段（核心表需审批）
ALTER TABLE `t_user_info` 
DROP COLUMN `old_field`;
```

## 六、AI 编写数据库代码 Prompt 模板

### 6.1 建表 Prompt

```text
【任务】生成建表 SQL 和 Entity 类

【数据库】MySQL 8.0.33（端口 3307）
【表名】t_xxx
【库名】gov_xxx
【业务说明】XXX业务表

【字段要求】
1. 必须包含必备字段：id（BIGINT雪花ID）、create_time、update_time、create_by、update_by、deleted
2. deleted 字段默认值 0，逻辑删除
3. 字符集 utf8mb4，排序规则 utf8mb4_unicode_ci
4. 高频查询字段必须加索引
5. 唯一编码字段必须加唯一索引
6. 敏感字段（身份证、手机号、密码）必须标注 [脱敏]

【命名要求】
1. 表名：t_ + 业务名（下划线）
2. 字段名：小写字母+下划线
3. 索引名：idx_字段名 或 uk_字段名（唯一）

【输出】
1. 建表 SQL（CREATE TABLE IF NOT EXISTS）
2. MyBatis-Plus Entity 类（继承 BaseEntity，@TableName，@TableId ASSIGN_ID，@TableLogic）
```

### 6.2 Entity Prompt

```text
【任务】生成 MyBatis-Plus Entity 类

【技术栈】SpringBoot 2.7.18 + MyBatis-Plus 3.5.5
【表名】t_xxx
【字段】（列出字段）

【约束】
1. 继承 gov-common 中的 BaseEntity
2. @TableId 使用 IdType.ASSIGN_ID（雪花ID）
3. deleted 字段加 @TableLogic
4. 使用 Lombok @Data
5. 所有字段加 OpenAPI 3 @Schema 注解（不是 Swagger 2 @ApiModelProperty）
6. 敏感字段不加 @Schema 或标注"脱敏字段"
```

## 七、Docker 初始化执行顺序

Docker 容器启动时，按文件名顺序执行 `docker/mysql/init/` 目录下的 `.sql` 文件：

```text
01-init-db.sql       ← 创建所有数据库（nacos + 10个业务库）
02-gov_user.sql      ← 用户认证库建表（13张表）
03-gov_item.sql      ← 事项管理库建表（8张表）
04-gov_reception.sql ← 统一受理库建表（7张表）
05-gov_activiti.sql  ← 审批流转库建表（8张表）
06-gov_license.sql   ← 电子证照库建表（6张表）
07-gov_complaint.sql ← 投诉建议库建表（6张表）
08-gov_open.sql      ← 政务公开库建表（6张表）
09-gov_datashare.sql ← 数据共享库建表（6张表）
10-gov_message.sql   ← 消息通知库建表（5张表）
11-gov_monitor.sql   ← 监察审计库建表（7张表）
99-init-data.sql     ← 插入初始化数据（最后执行）
12-98                ← 预留序号，用于新增表或 ALTER 表
```

**总计：72 张表**（13+8+7+8+6+6+6+6+5+7）

**一键启动**：
```bash
cd docker
docker-compose up -d
```

**重建 MySQL**（表结构变更后）：
```bash
docker-compose down
docker-compose up -d mysql
```

## 八、常见错误及修正

| 错误 | 正确做法 |
|------|----------|
| `id INT AUTO_INCREMENT` | `id BIGINT NOT NULL`（雪花ID） |
| 缺少 `deleted` 字段 | 必须加 `deleted TINYINT DEFAULT 0` |
| 手动用 Navicat 改表 | 必须写 ALTER TABLE SQL 并提交 |
| 敏感字段无标注 | 必须加 `[脱敏]` 注释 |
| 索引名随意命名 | 必须用 `idx_` 或 `uk_` 前缀 |
| 表名无前缀 | 必须用 `t_` 前缀 |
| 使用 `java.util.Date` | 使用 `LocalDateTime` |
| 修改已有 SQL 文件 | 新建 ALTER 文件（XX-alter.sql） |

## 九、数据库变更审核清单

组长审核 SQL 时检查：

| 检查项 | 是否通过 |
|--------|----------|
| 表名以 `t_` 开头 | ✅/❌ |
| 包含必备字段（id、create_time、update_time、deleted） | ✅/❌ |
| 主键使用 BIGINT 雪花ID | ✅/❌ |
| 唯一编码有唯一索引 | ✅/❌ |
| 外键关联有索引 | ✅/❌ |
| 敏感字段标注 `[脱敏]` | ✅/❌ |
| 字符集 utf8mb4 | ✅/❌ |
| 有表注释 | ✅/❌ |
| ALTER SQL 不是重建表 | ✅/❌ |
| SQL 文件放在 docker/mysql/init/ | ✅/❌ |
| 使用未占用的序号（12-98） | ✅/❌ |
