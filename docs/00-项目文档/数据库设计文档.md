# 数据库设计文档

> 本文档定义所有微服务的数据库表结构规范，供组员开发参考，也供 AI 生成代码时遵循。**所有成员使用 AI 开发时必须按本文档约束生成数据库相关代码。**

---

## 一、全局设计原则

### 1.1 必须遵守的规则

| 规则 | 说明 | 示例 |
|------|------|------|
| 表名前缀 | 所有表以 `t_` 开头，后接业务模块缩写 | `t_user_info`、`t_item_base`、`t_reception_record` |
| 主键类型 | `BIGINT`，使用雪花 ID（Hutool `IdUtil.getSnowflakeNextId()`） | `id BIGINT PRIMARY KEY` |
| 必备字段 | 每张表必须包含：`id`、`create_time`、`update_time`、`create_by`、`update_by`、`deleted` | 见下方模板 |
| 逻辑删除 | `deleted` 字段，`0`=正常，`1`=删除，MyBatis-Plus 全局配置 | `@TableLogic` |
| 字符集 | `utf8mb4` + `utf8mb4_unicode_ci` | 支持中文、表情 |
| 时间字段 | `DATETIME` 或 `TIMESTAMP`，默认值 `CURRENT_TIMESTAMP` | `create_time DATETIME DEFAULT CURRENT_TIMESTAMP` |
| 状态字段 | `TINYINT`，必须配合枚举类说明 | `status TINYINT DEFAULT 0 COMMENT '0待审 1通过 2驳回'` |
| 索引命名 | `idx_字段名` 或 `idx_字段1_字段2`（联合索引） | `idx_user_id`、`idx_item_status` |

### 1.2 建表模板（所有表必须遵循）

```sql
CREATE TABLE `t_xxx_xxx` (
  `id` BIGINT NOT NULL COMMENT '主键（雪花ID）',
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `create_by` BIGINT COMMENT '创建人ID',
  `update_by` BIGINT COMMENT '更新人ID',
  `deleted` TINYINT DEFAULT 0 COMMENT '逻辑删除：0正常 1删除',
  -- 业务字段从这里开始
  `xxx` VARCHAR(100) COMMENT '字段说明',
  PRIMARY KEY (`id`),
  KEY `idx_xxx` (`xxx`)  -- 高频查询字段必须加索引
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='表说明';
```

### 1.3 MyBatis-Plus Entity 模板

```java
@Data
@TableName("t_xxx_xxx")
public class XxxEntity extends BaseEntity {
    
    @TableId(type = IdType.ASSIGN_ID)  // 雪花ID
    private Long id;
    
    // 业务字段
    private String xxx;
    
    @TableLogic
    private Integer deleted;
}

// BaseEntity（gov-common 中定义）
@Data
public class BaseEntity {
    private LocalDateTime createTime;
    private LocalDateTime updateTime;
    private Long createBy;
    private Long updateBy;
}
```

## 五、数据库变更流程

1. **新增表**：先在 `sql/schema/` 目录编写建表 SQL，再生成 Entity。
2. **修改表**：先修改 SQL 文件，再同步 Entity，并在 MR 中说明变更点。
3. **删除表**：逻辑删除字段 `deleted=1`，不物理删除数据。
4. **索引变更**：必须说明原因（查询频率、性能优化）。
