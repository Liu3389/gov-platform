## 一、各服务数据库设计

### 1.1 gov_user（用户认证库）— 组员A

**库名**：`gov_user`  
**Redis db**：1  
**端口**：8081

| 表名 | 说明 | 核心字段 |
|------|------|----------|
| `t_user_info` | 用户基本信息 | `username`、`password` [脱敏][BCrypt]、`phone` [脱敏]、`real_name`、`id_card` [脱敏][SM4]、`status` |
| `t_dept_info` | 部门信息 | `dept_name`、`dept_code`、`parent_id`、`dept_type`、`leader_id`、`status` |
| `t_role_info` | 角色信息 | `role_name`、`role_code`、`status` |
| `t_user_role` | 用户角色关联 | `user_id`、`role_id` |
| `t_menu` | 菜单信息 | `menu_name`、`menu_code`、`menu_url`、`menu_type`、`parent_id`、`sort` |
| `t_role_menu` | 角色菜单关联 | `role_id`、`menu_id` |
| `t_user_realname` | 实名认证记录 | `user_id`、`real_name`、`id_card` [脱敏][SM4]、`cert_type`、`verify_status` |
| `t_ca_certificate` | CA证书绑定 | `user_id`、`cert_sn`、`cert_content` [脱敏][加密]、`expire_time`、`status` |
| `t_login_log` | 登录日志 | `user_id`、`login_ip`、`login_time`、`login_type`、`login_status` |
| `t_dict_data` | 数据字典 | `dict_type`、`dict_code`、`dict_name`、`dict_value`、`sort` |
| `t_region` | 行政区划 | `region_code`、`region_name`、`parent_code`、`region_level` |
| `t_config` | 系统配置 | `config_key`、`config_value`、`config_type` |
| `t_file_info` | 文件信息 | `file_name`、`file_path`、`file_type`、`file_size` |

## 二、敏感数据处理规范

| 数据类型 | 存储方式 | 加密算法 | 展示方式 |
|----------|----------|----------|----------|
| 密码 | BCrypt 哈希 | BCrypt | 不展示 |
| 身份证号 | SM4 加密 | BouncyCastle SM4 | `320***********1234` |
| 手机号 | 明文存储 | — | `138****8000` |
| CA 证书内容 | SM4 加密 | BouncyCastle SM4 | 不展示 |
| 数据库密码 | SM4 加密 | BouncyCastle SM4 | 不展示 |

## 四、开发流程

1. **新增表**：先在 `sql/schema/` 目录编写建表 SQL，再生成 Entity。
2. **修改表**：编写 ALTER TABLE SQL，放到 `sql/schema/alter_xxx.sql`。
3. **删除表**：逻辑删除字段 `deleted=1`，不物理删除数据。
4. **索引变更**：必须说明原因（查询频率、性能优化）。

**详细规范请参考《数据库开发规范.md》。**
