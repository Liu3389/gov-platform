# 开发规范

> 适用于“智慧政务一体化便民服务平台”全项目，所有成员必须遵守。

---

## 一、工程结构规范

### 1.1 后端多模块结构

```text
gov-platform/
├── gov-platform-parent/         ← 父工程，统一版本管理
├── gov-common/                  ← 公共模块（组长维护，新增需申请）
├── gov-gateway/                 ← 网关服务（组长维护，端口：8080）
├── gov-user-service/            ← ① 用户认证服务（组员A，端口：8081）
├── gov-item-service/            ← ② 事项管理服务（组员A，端口：8082）
├── gov-reception-service/       ← ③ 统一受理服务（组员B，端口：8083）
├── gov-activiti-service/        ← ④ 审批流转服务（组员B，端口：8084）【核心】
├── gov-license-service/         ← ⑤ 电子证照服务（组员C，端口：8085）
├── gov-complaint-service/       ← ⑥ 投诉建议服务（组员C，端口：8086）
├── gov-open-service/            ← ⑦ 政务公开服务（组员D，端口：8087）
├── gov-datashare-service/       ← ⑧ 数据共享服务（组员D，端口：8088）
├── gov-message-service/         ← ⑨ 消息通知服务（组员A/B协助，端口：8089）
├── gov-monitor-service/         ← ⑩ 监察审计服务（组长维护，端口：8090）
└── gov-web/                     ← Vue 前端（组员D）
```

### 1.2 单个服务内部结构

```text
gov-user-service/
├── src/main/java/com/gov/user/
│   ├── UserApplication.java
│   ├── config/                   # 配置类
│   ├── controller/               # 控制层（必须加 Knife4j 注解）
│   ├── service/                  # 服务层
│   ├── mapper/                   # 数据访问层
│   ├── entity/                   # 实体
│   ├── dto/                      # 数据传输对象
│   ├── vo/                       # 视图对象
│   ├── enums/                    # 枚举
│   └── utils/                    # 本服务工具类
├── src/main/resources/
│   ├── application.yml
│   ├── application-dev.yml       # 本地统一配置，提交 Git
│   ├── application-local.yml     # 个人本地覆盖，不提交 Git
│   └── mapper/*.xml
└── pom.xml
```

### 1.3 gov-common 模块结构

```text
gov-common/
└── src/main/java/com/gov/common/
    ├── result/                   # Result<T> 统一返回
    ├── exception/                # 业务异常、全局异常处理
    ├── utils/                    # Redis、加密、脱敏、日期等工具
    ├── entity/                   # BaseEntity
    ├── enums/                    # 公共枚举
    ├── annotation/               # 公共注解
    ├── config/                   # MyBatis-Plus、Knife4j 等配置
    └── constant/                 # 常量
```

## 二、命名规范

### 2.1 包名 / 模块名

- 根包：`com.gov.{模块名}`，如 `com.gov.user`。
- 模块名：`gov-xxx-service`，禁止使用 `_` 或 `-` 以外的特殊字符。

### 2.2 类名 / 方法名 / 常量

| 类型 | 规范 | 示例 |
|------|------|------|
| 类名 | 大驼峰 | `UserController` |
| 方法名 | 小驼峰 | `getUserById` |
| 常量 | 全大写 + 下划线 | `MAX_RETRY_TIMES` |
| 布尔变量 | 避免 is 开头 | `deleted` 而非 `isDeleted` |
| 数据库表 | 小写 + 下划线 | `sys_user`、`t_apply_record` |
| 数据库名 | 小写 + 下划线 | `gov_user`、`gov_item` |
| 接口路径 | 小写 + 中划线（RESTful） | `/api/user/apply-records` |

## 三、数据库规范（MySQL 8.0.33）

### 3.1 数据库拆分

每人本地 MySQL 创建独立数据库，避免服务间表冲突：

```sql
CREATE DATABASE IF NOT EXISTS gov_user DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE DATABASE IF NOT EXISTS gov_item DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE DATABASE IF NOT EXISTS gov_reception DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE DATABASE IF NOT EXISTS gov_activiti DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE DATABASE IF NOT EXISTS gov_license DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE DATABASE IF NOT EXISTS gov_complaint DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE DATABASE IF NOT EXISTS gov_open DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE DATABASE IF NOT EXISTS gov_datashare DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE DATABASE IF NOT EXISTS gov_message DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE DATABASE IF NOT EXISTS gov_monitor DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE DATABASE IF NOT EXISTS nacos DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 3.2 字段设计

- 每张表必须包含：`id`、`create_time`、`update_time`、`create_by`、`update_by`、`deleted`（逻辑删除）。
- 主键使用 `BIGINT` 或 `VARCHAR(32)`，推荐使用雪花 ID（Hutool `IdUtil.getSnowflakeNextId()`）。
- 状态字段使用 `TINYINT` 并配合枚举说明。
- 严禁明文存储密码、身份证号等敏感信息，必须加密或脱敏。

### 3.3 Redis 多 db 使用

| db | 用途 |
|----|------|
| 0 | 公共缓存 |
| 1 | gov-user-service |
| 2 | gov-item-service |
| 3 | gov-reception-service |
| 4 | gov-activiti-service |
| 5 | gov-license-service |
| 6 | gov-complaint-service |
| 7 | gov-open-service |
| 8 | gov-datashare-service |
| 9 | gov-message-service |
| 10 | gov-monitor-service |

### 3.4 索引与 SQL 规范

- 高频查询字段必须建索引。
- 联合索引遵循最左前缀原则。
- 全部使用 MyBatis-Plus 或 XML，禁止字符串拼接 SQL。
- 复杂查询必须手写 `EXPLAIN` 检查执行计划。

## 四、工作流规范（Activiti 7.7.0）

- BPMN 文件统一放在 `src/main/resources/processes/`。
- 流程 Key 命名：`{业务缩写}_v{版本号}`，如 `apply_approval_v1`。
- 任务节点必须指定 `assignee` 或 `candidateGroups`。
- 流程变量统一使用常量类管理，禁止硬编码字符串。
- 每个流程必须有：启动、审批、驳回、转办、结束节点。

## 五、安全规范

### 5.1 接口安全

- 网关统一鉴权，内部服务互信但需校验来源 IP/Token。
- 对外接口使用 HTTPS + 双向 TLS（mTLS）。
- 接口请求加时间戳 + 签名字符串，防止重放攻击。
- 签名算法：HMAC-SHA256 或国密 SM2/SM3（BouncyCastle 1.77）。

### 5.2 CA 认证

- 服务端配置 CA 根证书信任库。
- 客户端调用时携带客户端证书。
- 证书私钥禁止上传 Git，使用 Docker Volume 挂载。

### 5.3 敏感数据

- 身份证号、手机号等展示必须脱敏（`138****8000`）。
- 日志中禁止打印密码、Token、身份证号。
- 配置文件中的密码使用 Jasypt 加密或环境变量注入。

## 六、日志与监控

- 统一使用 SLF4J + Logback。
- 日志级别规范：
  - `ERROR`：系统异常、需要立即处理。
  - `WARN`：业务异常、可容忍但需关注。
  - `INFO`：关键业务流程节点。
  - `DEBUG`：开发调试，生产关闭。
- 每个请求打印 `traceId`，方便链路追踪。
- 本地开发使用 `INFO`，测试/生产使用 `WARN` 及以上。

## 七、Git 协作规范

### 7.1 分支模型

| 分支 | 说明 | 保护级别 |
|------|------|----------|
| `main` | 稳定主分支 | 禁止直接 push，仅 MR/PR 合并 |
| `develop` | 日常开发集成分支 | 禁止直接 push，仅 MR/PR 合并 |
| `feature/*` | 功能分支 | 个人可 push |
| `bugfix/*` | 缺陷修复分支 | 个人可 push |
| `hotfix/*` | 紧急修复 | 组长操作 |

### 7.2 分支命名

```text
feature/{模块}-{功能简述}
bugfix/{模块}-{问题简述}
```

### 7.3 Commit Message

采用 Conventional Commits：

```text
feat(user): 增加手机号验证码登录

- 新增 SmsCodeService
- 集成 Redis 缓存验证码

AI-assisted: true
```

### 7.4 提交流程

```bash
git checkout develop
git pull origin develop
git checkout -b feature/user-phone-login
# 开发...
git commit -m "feat(user): xxx"
git fetch origin
git rebase origin/develop
git push origin feature/user-phone-login
# 发起 MR/PR → develop
```

## 八、本地环境配置管理

### 8.1 配置分层

| 文件 | 是否提交 Git | 用途 |
|------|--------------|------|
| `application.yml` | 是 | 通用默认配置 |
| `application-dev.yml` | 是 | 本地开发统一配置（端口、中间件地址） |
| `application-local.yml` | **否** | 个人本地覆盖（密码、调试开关） |

### 8.2 application-dev.yml 示例

```yaml
server:
  port: 8081

spring:
  application:
    name: gov-user-service
  datasource:
    url: jdbc:mysql://127.0.0.1:3306/gov_user?useUnicode=true&characterEncoding=utf8
    username: root
    password: ${MYSQL_PASSWORD:123456}
  redis:
    host: 127.0.0.1
    port: 6379
    password: ${REDIS_PASSWORD:123456}
    database: 1
  cloud:
    nacos:
      discovery:
        server-addr: 127.0.0.1:8848
      config:
        server-addr: 127.0.0.1:8848
```

## 九、文档要求

- 每个服务必须包含 `README.md`，说明启动方式和关键配置。
- 接口文档使用 Knife4j 自动生成，网关聚合所有服务文档。
- 数据库变更必须同步维护 `sql/schema/` 与 `docker/mysql/init/`。
- 工作流变更必须同步更新流程图 PNG/SVG。
