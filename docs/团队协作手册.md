# 智慧政务一体化便民服务平台 — 团队协作手册

> 本文档是 `project_rules.md` 的补充，面向**团队管理**而非 AI 代码生成。
> 主规则（AI 自动读）见 `.trae/rules/project_rules.md`

---

## 一、代码审查检查表

合并 PR 前逐条核对：

- [ ] Controller 有 `@Tag` 和 `@Operation`，参数有 `@Parameter` 或 `@Schema`
- [ ] 新增/修改操作有 `@Log` 注解
- [ ] 返回值是 `Result<T>` 或 `Result<PageResult<T>>`
- [ ] Entity 继承 `BaseEntity`，有 `@EqualsAndHashCode(callSuper = true)`
- [ ] Controller 入参使用 DTO（非 Entity），出参使用 VO（非 Entity）
- [ ] 查询条件包含 `.eq(Deleted, 0)`
- [ ] 使用 LambdaQueryWrapper，无字符串拼接
- [ ] 写操作有 `@Transactional(rollbackFor = Exception.class)`
- [ ] Feign 客户端有 `fallbackFactory`，泛型与接口一致
- [ ] Feign FallbackFactory 覆盖了接口全部方法
- [ ] 异常使用 `BusinessException`，无暴力 `RuntimeException`
- [ ] `mvn compile` 通过
- [ ] Knife4j 文档中新接口已出现（http://localhost:8091/doc.html）
- [ ] 敏感信息未硬编码
- [ ] 新增数据库表在 `docker/mysql/init/` 有对应 SQL
- [ ] 没有修改别人的模块文件

## 四、PR 提交流程

**PR 描述模板：**
```
## 改动模块
gov-xxx-service

## 改动内容
- 新增 xxx 接口
- 新增 xxx DTO / VO
- 新增 xxx 建表 SQL

## 验证结果
- [ ] mvn compile 通过
- [ ] Knife4j 上能查到新接口
- [ ] 本地自测通过

## AI 使用声明
AI-assisted: true（如果是 AI 生成）
```

**流程：** feature 分支开发 → `mvn compile` 通过 → 推远端 → PR → 组长 Review → 合并 main

## 六、跨模块变更申请流程

**任何影响其他成员的改动（数据库、公共模块、Feign 签名、VO 字段），必须先申请。**

### 申请表模板

```
【变更申请】
申请人：XXX
变更模块：gov-common / docker/mysql/init/ / gov-xxx-service
变更类型：加表 / 改表 / 新工具类 / 改Feign签名 / 改VO / 其他
变更内容：简要描述
影响范围：哪些人需要适配？
适配内容：受影响的人具体要改什么？
```

### 什么需要申请

| 变更 | 申请？ |
|------|:--:|
| 在自己模块加表/改表 | ✅ |
| 修改 gov-common 代码 | ✅（且只有组长能改） |
| 改 Feign 接口签名 | ✅ |
| 改 Controller 路径 | ✅ |
| 改已有 VO 字段 | ✅ |
| 在自己模块加新接口（不影响他人） | ❌ |

### 数据库变更流程

```
1. 发起人提交"变更申请表"
2. 组长审批 → 通知全队
3. 拉 sql-alter/xxx 分支 → 建 XX-alter.sql
4. PR → 组长合并
5. 全队 docker-compose down && docker-compose up -d mysql
```

> **永远不修改已有 SQL 文件**，用新 ALTER 文件。

### 公共模块变更流程（gov-common）

```
1. 成员提交"变更申请表"给组长
2. **组长自己写代码**（成员不能直接改 gov-common）
3. 组长合并 → 通知全队 "mvn install -pl gov-common"
```

### Feign 签名变更流程

```
1. 发起人提交申请表（列出所有调用方）
2. 组长确认调用方清单
3. 拉 cross/feign-xxx 分支，同时改被调方 Controller + 所有调用方 FeignClient
4. PR → 组长合并 → 通知全队
```

## 七、冲突预防

| 风险操作 | 预防措施 |
|----------|---------|
| 修改 Controller 路径 | 必须先通知所有调用方 |
| 修改 Feign 签名 | 必须先通知被调方确认 |
| 在 `gov-common/vo/` 加 VO | 告知其他人，避免同名冲突 |
| 修改 `gov-common` | 只有组长可以，成员提需求 |
| 新增 SQL 文件 | 用未占用的序号 |
| 修改 `project_rules.md` | 只有组长可以 |

## 八、组长职责

- [ ] 维护 `gov-common` + `gov-gateway`
- [ ] 审批所有 PR
- [ ] 维护 `project_rules.md`，通知全队
- [ ] 维护 Knife4j 网关聚合配置
- [ ] 每天检查 Knife4j 文档，确认各成员接口进度
- [ ] 保持网关白名单与业务接口同步

## 九、成员职责

- [ ] 每天启动自己服务 + 网关，在 Knife4j 上确认接口可见
- [ ] 遵循 `project_rules.md` 所有编码规范
- [ ] 写操作加 `@Transactional`
- [ ] Feign 有 fallbackFactory
- [ ] 新增表同步提交 SQL
- [ ] PR 前跑通 `mvn compile`
- [ ] 不碰别人模块代码
- [ ] 用 AI 时附上 `project_rules.md` 作为 Prompt
