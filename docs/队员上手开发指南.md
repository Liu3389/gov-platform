# 队员上手开发指南（从克隆到合并）

> **阅读时长**：10 分钟。读完就能开始用 AI 写代码。

---

## 一、第一天：环境搭建

```bash
# 1. 克隆仓库
git clone <仓库地址>
cd zwpt001

# 2. 启动中间件（MySQL、Redis、Nacos、RabbitMQ）
docker-compose -f docker/docker-compose.yml up -d

# 3. 验证中间件
docker ps  # 看到 5 个容器运行（mysql/redis/nacos/rabbitmq/nginx）

# 4. 创建自己的开发分支（以成员A为例）
git checkout -b feature/user-auth main

# 5. 在 IDE（Trae/Cursor）中打开项目
#    → IDE 会自动读取 .trae/rules/project_rules.md
#    → 再手动读自己模块的子规则：.trae/rules/tasks/controller.md 等
```

---

## 二、每天：启动服务

你只需要启动 **网关 + 自己负责的 2-3 个服务**，不用启全部 10 个。

| 你的角色 | 启动这些 |
|----------|---------|
| 成员A | gateway + user-service + item-service + monitor-service |
| 成员B | gateway + reception-service + open-service |
| 成员C | gateway + activiti-service + license-service |
| 成员D | gateway + complaint-service + datashare-service + message-service |

```
在 IDE 中依次 Run → GatewayApplication → 自己模块的 Application
打开 http://localhost:8091/doc.html 确认网关和自己模块的接口都在
```

## 三、用 AI 写代码的标准流程

### 3.1 给 AI 发指令

```
【任务类型】Controller              ← 告诉 AI 读哪个子规则
【项目规则】.trae/rules/project_rules.md + .trae/rules/tasks/controller.md
【我负责的模块】gov-user-service（端口：8081）
【需求】实现手机号验证码登录接口
  1. POST /user/login  参数 phone + smsCode  返回 LoginVO（含token）
  2. POST /user/sendSms 参数 phone  返回 void
【输入/输出】LoginDTO{phone, smsCode} → Result<LoginVO>
【参考已有代码】gov-item-service 的 ItemController

请给出 Controller + Service + ServiceImpl + LoginDTO + LoginVO 完整代码。
```

### 3.2 不同任务的 Prompt 头

| 任务类型 | Prompt 头 |
|---------|----------|
| 写 Controller | `【项目规则】.trae/rules/project_rules.md + .trae/rules/tasks/controller.md` |
| 写 Service | `【项目规则】.trae/rules/project_rules.md + .trae/rules/tasks/service.md` |
| 写 Entity/DTO/VO | `【项目规则】.trae/rules/project_rules.md + .trae/rules/tasks/entity.md` |
| 写 Feign 调用 | `【项目规则】.trae/rules/project_rules.md + .trae/rules/tasks/feign.md` |
| 写 YAML 配置 | `【项目规则】.trae/rules/project_rules.md + .trae/rules/tasks/yaml.md` |
| 写建表 SQL | `【项目规则】.trae/rules/project_rules.md + .trae/rules/tasks/sql.md` |

> **用 Trae/Cursor**：主规则自动注入，只需在对话里说"也读一下 .trae/rules/tasks/controller.md"
> **用网页版 AI**：把 project_rules.md + 对应 tasks/ 子规则的文件内容全复制到对话框开头

## 四、开发完成后：验证 + 提交

### 4.1 本地验证

```
1. mvn compile -pl gov-platform-parent    ← 编译全项目，确保不报错
2. 在 Knife4j(http://localhost:8091/doc.html) 上确认新接口出现
3. 用 Knife4j 的 Try It 功能 -> 把 Token 填到右上角 🔒 Authorize -> 测试接口
4. 用 AI 自动跑测试（推荐）：
   【任务类型】API 测试
   【项目规则】.trae/rules/project_rules.md + .trae/rules/tasks/testing.md
   【我负责的模块】gov-user-service（端口：8081）
   请按测试规则测试我模块的所有接口，输出测试报告。
   → AI 会自动生成 Token、逐个调接口、给你测试报告
```

### 4.2 提交代码

```bash
git add gov-user-service/src/main/java/com/gov/user/   # 只加自己的模块
git commit -m "$(cat <<'EOF'
feat(user): 增加手机号验证码登录

- 新增 SmsCodeService 发送验证码
- 验证码存入 Redis，60秒内不可重复发送
- 登录接口新增 smsCode 参数

AI-assisted: true
EOF
)"
git push origin feature/user-auth
```

### 4.3 创建 PR

在 GitHub/GitLab 上创建 Pull Request → 填写 PR 描述模板：

```
## 改动模块
gov-user-service

## 改动内容
- 新增 /user/login 接口
- 新增 /user/sendSms 接口
- 新增 LoginDTO、LoginVO
- 新增 SmsCodeService

## 验证结果
- [x] mvn compile 通过
- [x] Knife4j 上能查到新接口
- [x] 本地自测通过

## AI 使用声明
AI-assisted: true
```

然后通知组长 Review。

## 五、遇到各种情况怎么办

### 情况A：我加了新表

1. 提交**变更申请表**给组长（见 `docs/团队协作手册.md` 第六章）
2. 组长审批后 → 拉 `sql-alter/xxx` 分支
3. 在 `docker/mysql/init/` 下新建 `12-xxx.sql`（用未占用的序号 12-98）
4. 提交 PR → 组长合并 → 通知全队重建 MySQL

```
【变更申请】
申请人：成员A
变更模块：docker/mysql/init/
变更类型：加表
变更内容：在 gov_user 库加 t_sms_log 表
影响范围：无
适配内容：无
```

### 情况B：我要改已有表的字段

1. 提交申请表给组长
2. 审批后 → 拉 `sql-alter/xxx` 分支
3. 新建 `13-alter-xxx.sql`（**决不修改已有的 .sql 文件**）
4. 内容用 ALTER TABLE 语句
5. PR → 合并 → 全队 `docker-compose down && docker-compose up -d mysql`

### 情况C：我要调别人服务的接口

1. 先去 `http://localhost:8091/doc.html` 看对方的接口是否已上线
2. 如果没上线 → 找对方催
3. 上线了 → 写 FeignClient + FallbackFactory（跟着 `.trae/rules/tasks/feign.md` 写）
4. 告诉你调的那个人"我接上你的 /xxx 接口了"

### 情况D：我要改自己 Controller 的路径

1. 提交申请表给组长（列出所有调用方）
2. 组长确认调用方清单
3. 别自己改 → 组长会创建 `cross/feign-xxx` 分支同时改你的 Controller + 调用方的 FeignClient
4. 合并后组长通知所有人

### 情况E：我需要 gov-common 加新工具类

1. 提交申请表给组长，描述你需要什么功能
2. **组长自己写**（你不能改 gov-common）
3. 组长合并后通知你 `mvn install -pl gov-common`
4. 你 pull 最新代码后继续开发

### 情况F：我的 PR 被组长打回了

1. 对照 `docs/团队协作手册.md` 第一章的检查清单逐条排查
2. 最常见打回原因：没加 `@Transactional`、Feign 没写 fallbackFactory、Entity 直接暴露
3. 修复后推送到同一分支 → PR 自动更新

### 情况G：pull 了最新代码后编译报错

1. `mvn install -pl gov-common`（gov-common 可能更新了）
2. 如果还报错 → 检查是不是少建了表 → `docker-compose down && docker-compose up -d mysql`
3. 还是不行 → 找组长

### 情况H：启动服务时报端口冲突

```
lsof -i :8081    ← 看谁占了你的端口
kill <PID>       ← 干掉它
```

如果 8091 被占 → 组长改了网关端口，不需要你处理。

### 情况I：Docker 容器挂了

```
docker ps -a                          ← 看哪些容器没运行
docker-compose -f docker/docker-compose.yml up -d   ← 重启所有中间件
docker logs gov-mysql --tail 20       ← 看容器日志排查错误
```

常见问题：
- MySQL 起不来 → `rm -rf docker/mysql_data` 清理数据 → 重建
- Nacos 起不来 → 检查 8848 端口是否被占
- RabbitMQ 起不来 → 检查 5672 端口是否被占

### 情况J：需要用自己的本地配置（不改公共配置）

```
不要改 application-dev.yml ← 这是共享的！
复制 application-local.yml 改为自己的配置
启动时加 profile：--spring.profiles.active=dev,local
application-local.yml 不提交 Git（已在 .gitignore）
```

### 情况K：IDEA 打开项目后全是报错

```
1. File → Project Structure → SDK 选 JDK 17
2. 右键 gov-platform-parent/pom.xml → Maven → Reload Project
3. mvn clean compile -pl gov-platform-parent
```

## 六、日常自查清单（每天提交前）

- [ ] `mvn compile -pl gov-platform-parent` 通过
- [ ] 写了 `@Tag` / `@Operation` / `@Parameter` / `@Schema`
- [ ] 写操作加了 `@Log` 和 `@Transactional`
- [ ] 入参是 DTO（不是 Entity），出参是 VO（不是 Entity）
- [ ] Feign 有 fallbackFactory
- [ ] 查询有 `.eq(Deleted, 0)`
- [ ] 用了 LambdaQueryWrapper，没用字符串拼接
- [ ] 用了 `LocalDateTime`，没用 `Date`
- [ ] 只改了负责的模块，没动别人文件
- [ ] 新接口在 Knife4j 上可见
- [ ] AI 生成的代码标了 `AI-assisted: true`

## 七、快速参考卡

| 场景 | 怎么做 |
|------|--------|
| 今天开始开发 | `docker-compose up -d` → 启动网关+自己模块 → 开 Knife4j |
| 用 AI 写 Controller | Prompt 头加 `tasks/controller.md` |
| 用 AI 写 SQL | Prompt 头加 `tasks/sql.md` |
| 本地编译 | `mvn compile -pl gov-platform-parent` |
| 测接口 | Knife4j Try It + Token（找组长要） |
| 提交代码 | `git commit -m "feat(module): xxx"` + `AI-assisted: true` |
| 加表/改表 | 提交申请表 → `sql-alter/` 分支 → 新建 XX.sql |
| 调别人接口 | 看 Knife4j → 写 FeignClient → 告知对方 |
| 改路径/签名 | 提交申请表 → 组长创建跨模块分支 |
| 编译报错 | `mvn install -pl gov-common` → 重建 MySQL |
| 看不懂怎么写 | 去 Knife4j 看已有接口 = 最好的参考代码 |
